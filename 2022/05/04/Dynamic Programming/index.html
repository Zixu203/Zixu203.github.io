Prologue

What is Dynamic Programming?

Details

Dynamic Programming is a technique in computer programming that helps to efficiently solve a class of problems that have overlapping subproblems and optimal substructure property.

If any problem can be divided into subproblems, which in turn are divided into smaller subproblems, and if there are overlapping among these subproblems, then the solutions to these subproblems can be saved for future reference. In this way, efficiency of the CPU can be enhanced. This method of solving a solution is referred to as dynamic programming.

Such problems involve repeatedly calculating the value of the same subproblems to find the optimum solution.

53. Maximum Subarray

Details

The key to solve this question is you need to relate the different situation with previous result.
For example, what is the maximum value of a continuous subarray when \({nums[i]}\) is its last element?

\({ ans=\left\{ \begin{array}{ll} \color{red}{[-2]} & =-2 & {, if\ \ } i=0\\ [1] & =\ \ \ 1 & {, if\ \ } i=1\\ \color{red}{[-2, 1]} & =-1 & {, if\ \ }i=1\\ [-3] & =-3 & {, if\ \ } i=2\\ \color{red}{[1, -3]} & =-2 & {, if\ \ } i=2\\ [-2, 1, -3] & =-4 & {, if\ \ } i=2\ \end{array}\right. }\)

Apparently, the result \({[-2, 1]}\) is add 1 after solution \({i=0}\), now we can list some basic calculation's rules:

  • \({a+b}\) will be smaller than \({a}\) if \({b<0}\)
  • \({a+0}\) will not be bigger than \({a}\)

In conclusion, if \({nums[i-1]\leq0}\), then abandon the previous result. Else, \({newSum=sum+nums[i]}\)
Having said that, now we know the state transition equation in this question:

\({ \large{ dp[i]=\left\{ \begin{array}{ll} nums[i] & {, if\ \ } nums[i-1] \leq{} 0 \\ dp[i-1]+nums[i] & {, if\ \ } nums[i-1]>0\ \end{array}\right. } \\ }\)

In this question, we only need the maximum, so we can rewrite the equation: \({ \large{dp[i]=max(nums[i], dp[i-1]+nums[i])} \\ }\)

918. Maximum Sum Circular Subarray

Details

In this question, we can divided to two cases:
Case 1: The subarray is in the middle part.
Case 2: The subarray take a part of head array and tail array.
But we can tranform case 2 into case 1, which mean we use the total sum of the whole array to minus the middle part.
Here is the diagram:

\({Proof:}\)

\({\ \ \ max(prefix+suffix)}\)

\({= max(totalSum-subArray)}\)

\({= totalSum + max(-subArray)\ \ \ \color{Gray}{// totalSum\ is\ a\ constant}}\)

\({= totalSum + min(subArray)}\)

\({Corner Case:}\) If all numbers are negative, \({maxSum=max(nums)}\) and \({minSum = sum(nums)}\)
In this case, \({max(maxSum, total-minSum)=0}\)
Thus, we need to return the \({maxSum}\) instead of 0.

Finally, we get the state transition equation:

\({ \large{ ans=\left\{ \begin{array}{ll} max(maxSum,\ total-minSum) & {, if\ \ } maxSum>0 \\ maxSum & {, if\ \ } maxSum \leq 0 \end{array} \right. } \\ }\)
